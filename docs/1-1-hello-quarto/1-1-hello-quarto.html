Quarto formats

One install, “Batteries included”

  • RMarkdown grew into a large ecosystem, with varying syntax.

  • Quarto comes “batteries included” straight out of the box

    • HTML reports and websites
    • PDF reports
    • MS Office (Word, Powerpoint)
    • Presentations (Powerpoint, Beamer, revealjs)
    • Books
  • Any language, exact same approach and syntax

One syntax, many formats

  • Documents: HTML, PDF, Word
  • Presentations: revealjs, beamer, pptx
  • Multi-page: websites, blogs, books
  • Specialised: journal articles, dashboards, interactive (Shiny)

Any language: R, Python, Julia, Observable. Same syntax, same approach.

Your turn

In your exercises project in RStudio on Posit Cloud, go to File > New File > Quarto document to create a Quarto document with HTML output. Save the file as my-first-document.qmd, then render. Use the visual editor for the next steps. Save (Cmd/Ctrl+S) before each render.

  • In the YAML header at the top of the document, add a title in the title: field and your name to the author: field.

  • Create three sections in the document formatted as Heading 2, name these sections

    • Joyful tasks
    • Challenging tasks
    • A fact to learn

  • Stretch goal: In the YAML header add a table of contents in the format: field. Tip: check quarto.org and use search with “HTML table of contents”.

  • Stretch goal: Change the html theme to sketchy. Tip: check quarto.org and use search with “HTML theming”.

  • Before you turn back: without looking, what does the toc: field do? What is the difference between visual and source editor mode?
